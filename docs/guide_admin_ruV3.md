# Руководство администратора. V3

Проект «Проверка магистерских проектов»

Дата: 06.05.2026

---

# 1. О проекте

Для автоматизации отдельных этапов аттестации магистрантов используется Telegram-бот `@magistrcheckbot`.

Система предназначена для:

* регистрации участников проекта;
* проверки промежуточных отчетов;
* проверки ссылок на материалы магистерского проекта;
* анализа диссертаций;
* формирования справок и отчетов;
* дистанционного контроля со стороны научных руководителей и администраторов.

---

# 2. Архитектура системы

Система состоит из:

* Telegram-бота;
* ядра обработки;
* Google Таблицы;
* Google Docs API;
* Google Drive API.

Google Таблица является основным источником данных (source of truth).

---

# 3. Основные роли

## Магистрант

Может:

* зарегистрироваться;
* запускать проверку проекта;
* получать справку о состоянии проекта.

---

## Научный руководитель

Может:

* просматривать состояние проектов своих магистрантов;
* получать список магистрантов;
* запускать проверку по фамилии магистранта.

---

## Администратор

Может:

* просматривать любые проекты;
* отправлять замечания магистрантам;
* формировать PDF-карточки;
* выполнять ручной контроль.

---

# 4. Основной пользовательский сценарий

## Шаг 1. Идентификация

Пользователь запускает:

```text
/start
```

Бот:

1. проверяет Telegram ID;
2. определяет роль;
3. при необходимости запрашивает ФИО;
4. выполняет поиск по Google Таблице;
5. привязывает Telegram ID;
6. показывает меню.

---

## Шаг 2. Проверка проекта

Команда:

```text
/справка
```

запускает:

* проверку промежуточного отчета;
* проверку ссылок;
* анализ диссертации;
* формирование отчета.

Повторный запуск `/справка` выполняет повторную проверку проекта.

---

# 5. Команды

Документ `commands.md` является canonical source для Telegram-команд.

Если руководство противоречит `commands.md`, приоритет имеет `commands.md`.

---

| ------------ | -------------------- |
| `/start`     | запуск системы       |

| Команда            | Назначение          |
| ------------------ | ------------------- |
| `/справка Фамилия` | проверка проекта    |
| `/список`          | список магистрантов |

---

## Команды администратора

| Команда            | Назначение          |
| ------------------ | ------------------- |
| `/справка Фамилия` | проверка проекта    |
| `/список`          | список магистрантов |
| `/ошибка`          | отправка замечания  |

---

# 6. Команда `/ошибка`

Пример:

```text
/ошибка 3, 19, 38
```

После запуска:

1. администратор вводит текст замечания;
2. бот показывает предпросмотр;
3. доступны кнопки:

   * Отправить
   * Отредактировать
   * Отмена
4. сообщение отправляется магистрантам;
5. магистрантам предлагается повторно выполнить `/справка`.

---

# 7. Этапы проверки

## Этап 1. Проверка промежуточного отчета

Проверяется:

* доступность ссылки;
* тип ссылки;
* наличие фразы «Промежуточный отчет».

---

## Этап 2. Проверка ссылок проекта

Проверяются:

* папка проекта;
* заключение ЛКК;
* диссертация;
* публикация.

---

## Этап 3. Анализ диссертации

Определяются:

* число страниц;
* число источников;
* соответствие оформлению.

---

## Этап 4. Извлечение данных диссертации

Определяются:

* название диссертации;
* язык диссертации.

---

# 8. Ручной ввод ссылок

Если Google Docs API не смог извлечь ссылки автоматически, администратор может вручную внести URL в лист «Регистрация».

После сохранения администратор повторно запускает:

```text
/справка Фамилия
```

---

# 9. Нормализация групп

На вкладке «Магистранты» используется поле:

```text
Группа норм
```

Администратор вручную указывает стандартизированное название группы.

---

# 10. Snapshot Architecture

Система может сохранять JSON-снимки проекта.

Snapshot содержит:

* метаданные;
* результаты проверок;
* ссылки;
* статусы этапов.

Snapshot может использоваться:

* для повторного формирования отчетов;
* для PDF-карточек;
* для восстановления состояния проверки.

---

# 11. CLI-only операции

Следующие операции не должны использоваться через Telegram UI:

* dashboard sync;
* массовые рассылки;
* maintenance scripts;
* repair scripts;
* системная статистика.

---

# 12. Legacy-команды

Удалены:

* `/registr`
* `/recheck`
* `/cancel`

Повторная проверка выполняется через:

```text
/справка
```
