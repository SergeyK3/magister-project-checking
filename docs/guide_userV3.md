# Руководство пользователя Telegram-бота проверки магистерских проектов (V3)

## Назначение

Бот предназначен для:

* регистрации магистрантов;
* привязки Telegram-аккаунта к строке в таблице;
* запуска проверки проекта;
* получения краткой справки по проекту;
* взаимодействия научных руководителей и администраторов.

---

# Основной принцип работы

После команды:

```text
/start
```

бот автоматически:

1. определяет вашу роль по Telegram ID;
2. показывает меню вашей роли;
3. запускает нужный сценарий.

Для каждой роли существует один основной пользовательский путь:

* магистрант;
* научный руководитель;
* администратор.

---

# Магистрант

## Первый вход

Нажмите:

```text
/start
```

Если Telegram ID ещё не зарегистрирован:

* бот предложит выбрать роль;
* затем начнётся регистрация.

Во время регистрации:

* часть полей можно пропускать;
* Telegram ID подставляется автоматически.

---

## Команды магистранта

### Основная команда

```text
/start
```

Используется для:

* регистрации;
* продолжения незавершённой анкеты;
* открытия меню.

---

### Проверка проекта

Кнопка:

```text
Справка / проверка
```

или команда:

```text
/справка
```

Запускает:

* повторную проверку проекта;
* обновление краткой справки.

Если клиент Telegram не принимает кириллицу, используйте:

```text
/spravka
```

---

### Пропуск поля

Во время анкеты можно отправить:

```text
-
```

или:

```text
/skip
```

---

### Прерывание диалога

Для выхода из текущего сценария:

```text
/выход
```

---

# Научный руководитель

После:

```text
/start
```

бот показывает меню научного руководителя.

Доступные действия:

* проверка магистранта;
* просмотр зарегистрированных;
* просмотр незарегистрированных;
* получение справки;
* помощь.

Для некоторых действий бот дополнительно запросит:

* ФИО;
* номер строки;
* или выбор подопечного.

---

# Администратор

После:

```text
/start
```

администратор получает расширенное меню.

Основные возможности:

* запуск проверок;
* формирование карточек проекта;
* отправка сообщений;
* групповые напоминания;
* служебные действия;
* диагностические команды;
* операторская поддержка.

---

# Повторная проверка проекта

Во время проверки бот:

* сравнивает текущие входы;
* анализирует изменения;
* может использовать быстрый режим only-if-changed.

Если изменений нет:

* листы и история проверок не обновляются;
* бот сообщает, что изменения не обнаружены.

---

# Возможные сообщения

## «Без изменений»

Означает:

* с прошлой проверки входы не изменились;
* повторная обработка не запускалась.

---

## Временное ограничение Google

Иногда Google временно ограничивает количество запросов.

В этом случае:

* подождите 30–60 секунд;
* затем снова нажмите:

```text
/start
```

или:

```text
/справка
```

---

# Кнопочные меню

После перезапуска бота Telegram может обновить:

* список команд;
* клавиатуру кнопок.

Если меню не появилось:

1. закройте чат;
2. откройте снова;
3. выполните:

```text
/start
```

---

# Важные ограничения

Пользователь:

* не может изменять чужие строки;
* не видит внутренние диагностические данные;
* не имеет доступа к служебным журналам.

Полный доступ:

* только у администраторов.

---

# Рекомендации

Рекомендуется:

* использовать кнопочное меню;
* не запускать несколько сценариев одновременно;
* завершать текущий диалог перед новым действием.

---

# Если возникла проблема

Попробуйте:

1. выполнить:

```text
/выход
```

2. затем снова:

```text
/start
```

Если проблема сохраняется:

* обратитесь к куратору или администратору.
