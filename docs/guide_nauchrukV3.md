# Руководство научного руководителя. V3

## Работа с ботом

Все действия выполняются в личном чате с ботом.

---

# Первый запуск

Команда:

```text
/start
```

запускает:

* идентификацию;
* определение роли;
* привязку Telegram ID.

---

# Основные возможности

Научный руководитель может:

* контролировать проекты магистрантов;
* получать информацию о состоянии проектов;
* просматривать списки магистрантов.

---

# Команды

| Команда            | Назначение           |
| ------------------ | -------------------- |
| `/start`           | запуск системы       |
| `/status Фамилия`  | проверка проекта подопечного |
| `/unreg`           | кто не зарегистрировался |
| `/reg_list`        | кто зарегистрировался |
| `/about`           | информация о проекте |

---

# Команда `/status`

Пример:

```text
/status Иванов
```

Команда запускает:

* повторную проверку проекта;
* анализ ссылок;
* формирование справки.

---

# Команды списков

Команда:

```text
/unreg
```

показывает подопечных, которые ещё не зарегистрировались.

Команда:

```text
/reg_list
```

показывает подопечных, которые уже зарегистрировались.

В списках используются данные листов `Магистранты` и `Регистрация`.

---

# Команда `/about`

Показывает ссылку на описание проекта:

```text
https://docs.google.com/document/d/1Dq0MJL1hrdNG4XYo-BNKMrmRfnO3-ndeTo42vg1GjUM/edit?usp=sharing
```

---

# Что показывают списки

Списки помогают контролировать:

* список магистрантов;
* статус регистрации;
* контактные данные.

---

# Работа с Google Таблицей

Основные вкладки:

* Магистранты
* Регистрация

---

# Важно

Если магистрант не зарегистрировался:

1. свяжитесь с ним;
2. попросите выполнить:

```text
/start
```
