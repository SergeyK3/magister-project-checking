# FSM

## Общие принципы

* FSM должен быть минималистичным;
* пользователь не должен помнить сложные команды;
* повторный `/start` всегда сбрасывает FSM;
* `/выход` очищает текущее состояние;
* pipeline проверки должен быть идемпотентным.

---

# STATE_UNREGISTERED

Пользователь еще не идентифицирован.

## Вход

* первый запуск бота;
* неизвестный Telegram ID.

## Переходы

```text
/start → STATE_WAITING_FIO
```

---

# STATE_WAITING_FIO

Ожидание ввода ФИО.

## Действия

Бот:

* получает ФИО;
* выполняет поиск по Google Таблице;
* учитывает казахско-русские варианты написания.

## Возможные результаты

### Пользователь найден

```text
→ STATE_ROLE_DETECTED
```

### Пользователь не найден

```text
→ STATE_ACCESS_DENIED
```

---

# STATE_ROLE_DETECTED

Роль пользователя определена.

## Возможные роли

* magistrant
* supervisor
* admin

## Действия

* Telegram ID сохраняется;
* выбирается соответствующее меню.

## Переходы

```text
→ STATE_MAIN_MENU
```

---

# STATE_MAIN_MENU

Главное меню пользователя.

## Доступные действия

### Магистрант

```text
/справка
```

### Научный руководитель

```text
/справка Фамилия
/список
```

### Администратор

```text
/справка Фамилия
/список
/ошибка
```

---

# STATE_RUNNING_CHECK

Выполняется pipeline проверки.

## Этапы

1. Проверка промежуточного отчета
2. Проверка ссылок
3. Анализ диссертации
4. Извлечение метаданных
5. Формирование отчета

## Свойства

* допускается повторный запуск;
* pipeline должен быть идемпотентным;
* ошибки API должны быть recoverable.

## Переходы

```text
→ STATE_SHOW_REPORT
```

---

# STATE_SHOW_REPORT

Показ результатов проверки.

## Доступные действия

* повторный `/справка`;
* `/выход`.

---

# STATE_WAITING_ADMIN_MESSAGE

Администратор вводит замечание.

## Вход

```text
/ошибка
```

## Действия

* ввод текста;
* предпросмотр;
* подтверждение.

## Переходы

```text
→ STATE_CONFIRM_ADMIN_MESSAGE
```

---

# STATE_CONFIRM_ADMIN_MESSAGE

Подтверждение отправки замечания.

## Кнопки

* Отправить
* Отредактировать
* Отмена

---

# STATE_SENDING_ADMIN_MESSAGE

Отправка замечания магистрантам.

## Действия

* отправка сообщений;
* предложение повторно выполнить `/справка`.

---

# STATE_ACCESS_DENIED

Пользователь не найден в системе.

## Сообщение

```text
Пользователь не найден.
Обратитесь к научному руководителю.
```

## Доступные действия

```text
/start
```
