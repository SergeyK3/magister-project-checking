# Commands

Технический перечень slash-команд и compatibility routes. Основной пользовательский UX описан в `docs/guide_admin_ruV3.md`: `/start` → определение роли → меню кнопками.

Соответствует регистрации хендлеров в `magister_checking/bot/app.py` и сценариям в `magister_checking/bot/handlers.py`. Русские команды `/старт`, `/справка`, `/выход` обрабатываются через `MessageHandler` с regex в том же приложении.

---

## Общие принципы

* Личный чат с ботом: большинство команд работает только в private (`filters.ChatType.PRIVATE`).
* В группах `/start` обрабатывается отдельным хендлером с подсказкой перейти в личку.
* Часть команд латиницей — технические alias Bot API (`/spravka`, `/cancel`, `/register`, …).

---

## Общие команды

| Команда | Alias | Назначение |
| ------- | ----- | ---------- |
| `/старт` | `/start` | регистрация и вход в сценарий |
| `/справка` | `/spravka` | проверка / справка (зависит от роли) |
| `/выход` | — | выход из диалога (сообщение с regex, см. app.py) |
| `/cancel` | — | отмена внутри `ConversationHandler` |
| `/register` | — | регистрация (латинский вход в тот же FSM) |
| `/help` | — | подсказка по роли (`help_command`) |

В меню BotFather через API задаются только короткие пункты (`start`, `spravka`, `help`) — см. `default_bot_commands()`.

---

## Магистрант

Через FSM и сервисные хендлеры:

| Команда | Назначение |
| ------- | ---------- |
| `/справка` | канонический запуск проверки своей строки «Регистрация» |
| `/status` | краткий статус своей строки (ветка без прав научрука) |
| `/recheck` | повторная проверка своей строки; можно `/recheck quick` |

---

## Научный руководитель

| Команда | Назначение |
| ------- | ---------- |
| `/справка` | проверка **своей** строки в «Регистрация», если аккаунт там есть (как у магистранта) |
| `/status ФИО …` | проверка подопечного по ФИО (лист «Магистранты», связка с научруком) |
| `/unreg` | список незарегистрировавшихся магистрантов данного научрука |
| `/reg_list` | список зарегистрировавшихся |

Указать в `/справка номер_строки` или `/справка Чужое ФИО` могут **только администраторы** — см. `spravka_start`.

## Администратор

Топ-уровень (не Conversation):

| Команда | Назначение |
| ------- | ---------- |
| `/admin` | панель и подсказки по `/project_card`, `/student_message`, `/student_message_bulk` |
| `/stats` | статистика |
| `/ops_row` | диагностика строки |
| `/sync_dashboard` | синхронизация дашборда регистрации |
| `/sync_magistrants` | обновление листа магистрантов |
| `/status` | если пользователь не в ветке научрука — краткий статус своей строки «Регистрация» (как у магистранта; см. `status_command`) |
| `/unreg` / `/reg_list` | без аргументов — как у научрука, если пользователь научрук; с аргументом ФИО — превью отчёта для указанного научрука |
| `/recheck` | повторная проверка; админ может указать строку или ФИО в команде |

Через FSM (те же входы, что у магистранта, плюс админские сценарии):

| Команда | Назначение |
| ------- | ---------- |
| `/справка` | без аргументов — меню форматов; с аргументом — проверка указанной строки/ФИО |
| `/spravka` | то же |
| `/project_card` | PDF-карточка по строке или ФИО |
| `/student_message` | напоминание одному магистранту |
| `/student_message_bulk` | групповое стандартное напоминание по номерам строк |

Также администратор может отправить документ `.json` (snapshot) в личку — см. `on_project_snapshot_json_file`.
