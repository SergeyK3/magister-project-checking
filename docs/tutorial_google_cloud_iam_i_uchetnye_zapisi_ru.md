# Учебно: Google — какой аккаунт для чего (консоль vs бот/CLI)

**Зачем этот файл:** чтобы не путать «под кем я в браузере» и «каким ключом ходит проверка в таблицы», и не перепроверять это каждый раз с нуля.

---

## 1. Две разные сущности

| Что | Зачем | Где смотреть |
|-----|--------|----------------|
| **Ваш человеческий Google-аккаунт** | Вход в **Google Cloud Console**: включить API, IAM, биллинг, политики org | Аватар в правом верхнем углу консоли |
| **Service Account (сервисный аккаунт)** | **Бот**, **CLI** (`check-row`, скрипты с `gspread`): доступ к **Sheets/Drive** по JSON-ключу | Файл из `GOOGLE_SERVICE_ACCOUNT_JSON` в `.env`; внутри поля `client_email`, `project_id` |

Это **не** «войти в Google под сервисным аккаунтом» в браузере. Скрипт **не использует** вкладку Chrome — он читает **JSON** и ходит в API от имени `client_email`.

---

## 2. Какой аккаунт нужен в консоли

- Проект **MagisterChecker** (`project_id`: **magisterchecker**) лежит в org **zdravconsult.com**.
- Чтобы **видеть** проект и **включать API**, в консоли нужен Google-аккаунт, которому выдали роль на **этот проект** (часто рабочий `@zdravconsult.com`, не обязательно личный Gmail).
- Если видите **«You need additional access»** / нет `resourcemanager.projects.get` — вы в консоли **под другим** пользователем или без роли на проект. Иконка **Gemini (звёздочка)** к правам на проект **не относится**.

Подробности по SA и путям в консоли: `docs/handoff_gcp_service_account_2026-04-26.md`.

---

## 3. Какой «аккаунт» нужен для таблиц (Sheets)

1. **Sheets API / Drive API** должны быть **включены** для GCP-проекта, в котором **создан** сервисный аккаунт (в JSON — `project_id`). Иначе ошибка вида: *Google Sheets API has not been used in project … or it is disabled*.
2. Каждая **книга Google Sheets**, куда или откуда читает код, должна быть **расшарена** на email **`client_email`** из того же JSON с правом **редактора** (или читателя, если только чтение).

Без шаринга будет другая ошибка (часто про отсутствие прав к файлу), не путать с отключенным API.

---

## 4. Почему «проверки» внезапно ломаются

- **Один и тот же** `python` / `.env` → один и тот же SA. Смена **вкладки браузера** или **аккаунта в Chrome** **не меняет**, чем ходит `check-row`, пока вы не меняете `.env` или ключ.
- Если **раньше** `check-row` писал в лист, а **сейчас** падает с **403 про Sheets API** — проблема на стороне **GCP** (API / политика org), а не «не тот аккаунт в браузере».

---

## 5. Быстрая шпаргалка команд (из корня репозитория)

Копирование листа «научрук» из другой книги:

```powershell
.\.venv\Scripts\python.exe scripts/copy_supervisors_sheet.py
```

Проверка строки (обращается к той же таблице, что в `SPREADSHEET_ID`):

```powershell
.\.venv\Scripts\python.exe -m magister_checking.cli check-row --row 4
```

Имеет смысл использовать **один** интерпретатор (например `.venv`), чтобы не путаться с путями и зависимостями.
