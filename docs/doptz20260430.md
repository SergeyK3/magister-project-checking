# Техническое задание — MagistrCheck
Дата: 30.04.2026

## 1. Назначение системы
Система предназначена для:
- идентификации пользователей Telegram
- привязки Telegram ID к записи в Google Sheets
- разграничения ролей: администратор, научрук, магистрант
- управления статусами магистрантов

## 2. Источник данных
Google Sheets:
ID: 1RuAXZt9wAu3CQNn3GAL65ifyjykN5AnD8XFwnZ_v39s

### Листы:
- Магистранты: ФИО, Телефон, tg_id, статус, научрук
- Научруки: ФИО, Телефон, tg_id
- Админы: ФИО, tg_id

## 3. Логика работы
Команда /start → проверка tg_id

Если найден:
- определение роли → выдача меню

Если не найден:
- запуск регистрации (FSM)

## 4. FSM (состояния)
- STATE_START
- STATE_CHECK_TG
- STATE_WAIT_FIO
- STATE_FIO_FOUND
- STATE_FIO_NOT_FOUND
- STATE_WAIT_PIN_REQUEST
- STATE_WAIT_PIN_INPUT
- STATE_PIN_VERIFY
- STATE_PIN_ERROR
- STATE_REGISTER_SUCCESS
- STATE_BLOCKED

## 5. PIN-код
- длина: 4–6 цифр
- срок действия: 5 минут
- максимум 3 попытки
- хранение: hash (SHA256)

Таблица хранения:
verification_codes:
- phone
- pin_hash
- expires_at
- attempts
- tg_id

## 6. Запись tg_id
После успешной проверки:
- если магистрант → запись в лист Магистранты
- если научрук → запись в лист Научруки

## 7. Меню

### Научрук:
- список незарегистрированных
- список зарегистрированных
- статус магистранта

### Магистрант:
- первичная регистрация
- обновить статус

### Админ:
- просмотр пользователей

## 8. Логирование

Фиксируются:
- авторизация
- регистрация
- PIN попытки
- ошибки

Пример:
{
  "event": "pin_verify",
  "tg_id": 123456,
  "status": "failed"
}

## 9. Ошибки
- PIN истёк
- пользователь не найден
- превышение попыток

## 10. Ограничения
- 1 tg_id = 1 пользователь
- строгий поиск ФИО

## 11. MVP
- PIN можно выводить в лог
- хранение в памяти допустимо

## 12. Поток
/start → проверка tg_id → меню или регистрация → ввод ФИО → PIN → запись tg_id → меню
