# Architecture Notes

## Основная идея

Проект представляет собой Telegram-бота для автоматизации проверки магистерских проектов.

Система состоит из:

* Telegram Bot
* Google Sheets
* Google Docs API
* Google Drive API
* Core processing pipeline

---

## Основные роли

* магистрант
* научный руководитель
* администратор

---

## Основной сценарий

1. Пользователь запускает `/start`
2. Бот определяет роль
3. Пользователь получает меню
4. Команда `/справка` запускает проверку
5. Формируется отчет

---

## Принципы архитектуры

* минимальное количество команд;
* единый pipeline проверки;
* отсутствие дублирующих команд;
* FSM-first архитектура;
* Google Sheets как source of truth;
* retry через повторный запуск `/справка`.

---

## Удаляемые legacy-команды

* `/recheck`
* `/registr`
* `/admin`

---

## CLI-only функции

* dashboard sync
* массовые рассылки
